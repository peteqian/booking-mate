# Design Philosophy

## Product Aesthetic

Buching should use a restrained enterprise workspace aesthetic built on the shadcn-style design system primitives.

- Use the design system color tokens intentionally, with stronger color reserved for status, feedback, and important emphasis.
- Favor calm, operational, trustworthy screens over playful or decorative ones.
- Use subtle borders, light fills, and soft shadows rather than heavy outlines or saturated panels.
- Let sidebar navigation and card/table layouts organize domain features clearly.
- Make the UI feel like a productivity tool for running a business, not a marketing site.

## UI Hierarchy

Use typography and spacing as the primary tools for communicating hierarchy.

- Typography should make importance obvious through size, weight, line height, and contrast.
- Spacing should separate concepts, group related controls, and create a clear reading path.
- Avoid relying on borders or dense containers as the default way to organize every section.

## Elevation

Use box shadow to show elevation when a surface should feel raised above the page.

- Prefer subtle shadows for cards, dialogs, popovers, and important workspace surfaces.
- Use stronger shadows only when the interaction requires clear foreground emphasis.
- Keep elevation consistent so similar surfaces feel like they belong to the same system.

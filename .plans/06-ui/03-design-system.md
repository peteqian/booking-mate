# Design System

## Scope

Use the existing Tailwind setup and local component conventions.

## Components Needed

- Button.
- Input.
- Textarea.
- Select.
- Dialog.
- Tabs.
- Badge.
- Table.
- Card.
- Toast.
- Dropdown menu.
- Sheet/drawer.
- Calendar integration styles.

## Product UI Principles

- Operational clarity over marketing gloss inside the dashboard.
- Public booking pages should feel polished and trustworthy.
- Avoid generic card grids when a table, timeline, or calendar communicates better.
- Mobile public registration must be first-class.

## States

Every major page needs:

- Loading skeleton.
- Empty state.
- Error state.
- Permission-denied state where relevant.

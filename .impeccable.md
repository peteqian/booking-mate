## Design Context

### Users

Booking Mate is used by teams that run classes, workshops, programs, sessions, and scheduled offerings. Users are operators: owners, admins, managers, and staff who need to create events, track status, manage registrations, and later coordinate resources, attendees, payments, and public booking pages.

### Brand Personality

Calm, operational, trustworthy. The product should feel like a capable business workspace, not a marketing site or playful consumer app.

### Aesthetic Direction

Use a restrained enterprise workspace aesthetic built on shadcn-style primitives and tokens. Reference direction: compact legal/finance/research SaaS workspaces with clear sidebar navigation, quiet surfaces, strong typography, disciplined spacing, and subtle elevation. Avoid generic GPT/shadcn card mosaics, overly sharp corners, loud gradients, ornamental color, and default SaaS dashboards.

### Design Principles

- Use typography and spacing first to create hierarchy.
- Preserve a comfortable default border radius across app surfaces.
- Use soft box shadows for elevated work surfaces, cards, dialogs, and active interaction states.
- Organize domain features through sidebar navigation, table layouts, kanban lanes, and operational panels.
- Reserve stronger color for status, feedback, and primary actions; keep routine workspace UI calm.
- Prefer distinctive composition over stacked generic cards: watermarks, rails, table-first density, and clear workspace regions are encouraged when they improve scanning.
